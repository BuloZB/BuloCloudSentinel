Subresource Integrity Example

This page demonstrates how to use Subresource Integrity (SRI) to ensure that resources loaded from external sources have not been tampered with.

How SRI Works

  1. Generate a cryptographic hash of the resource (SHA-384 recommended)
  2. Add the hash to the integrity attribute of the resource tag
  3. Add the crossorigin="anonymous" attribute to the resource tag
  4. The browser will verify the hash before executing the resource

When to Use SRI

  • When loading resources from CDNs
  • When loading resources from third-party sources
  • When you want to ensure that resources have not been tampered with
